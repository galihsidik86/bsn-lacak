05 · Stack
B BSN LACAK
Lapis keamanan

Defense-in-depth dari edge hingga query

1 Edge & transport

TLS di Caddy (Let's Encrypt otomatis), HSTS, CSP default-src 'none', rate-limit per IP (auth 10/15m, API 600/15m), CORS sempit ke WEB_ORIGIN.

2 Identitas & sesi

JWT akses 15 menit + refresh 7 hari di cookie httpOnly Secure SameSite=Strict. Refresh di-hash SHA-256, rotasi tiap pakai, reuse detection mencabut seluruh keluarga sesi.

3 Otorisasi & tenancy

Role-based (ADMIN / SUPERVISOR / PETUGAS) + row-level branch tenancy di setiap query. ADMIN HQ pakai header x-branch-id untuk berpindah cabang.

4 Aplikasi & upload

Validasi Zod di semua endpoint mutasi · magic-byte check + 8 MB/file · watermark server-side (sharp) sebelum simpan disk · rate-limit POST /kunjungan per user.

5 Observabilitas & audit

Pino dengan PII masking (nama nasabah, HP, koordinat tidak masuk log) · AuditLog untuk login/reuse/reassign/blast/auto-balance · retensi 365 hari + arsip.

6 Password & lockout

Policy 12+ karakter mix · bcrypt cost 12 · force-change first login · 5× salah → lock 15 menit · semua sesi dicabut saat ganti password.

Anti-fraud pipeline

Setiap laporan kunjungan diberi risk score sebelum disimpan

G GPS plausibility

Jarak antara titik upload dan alamat nasabah > 200 m → flag far-from-address. Mendeteksi laporan yang difile dari rumah / kantor.

E EXIF freshness

Bila DateTimeOriginal > 1 jam atau hilang → flag stale-photo. Memblok foto galeri lama yang diambil ulang.

S Speed jump

Pergerakan antar posisi GPS > 150 km/jam → flag impossible-speed. Indikasi spoofer / mock location.

W Watermark + magic byte

Header file diverifikasi (file-type), lalu sharp menempel watermark "BSN Lacak · petugas · nasabah · jam · GPS" — tidak mempercayai klien.

Offline-first

Tetap mencatat saat sinyal hilang

Service Worker + localStorage queue membuat aplikasi petugas berperilaku seperti aplikasi native — tidak ada laporan yang hilang gara-gara basement, lift, atau desa tanpa sinyal.

① Capture

Foto + GPS + form disimpan dulu ke localStorage queue. Form tahan tab-kill (Android camera intent tidak menghapus state).

② Queue

Setiap upload masuk antrian dengan retry exponential. Pengguna melihat badge "menunggu sinyal" di kartu kunjungan.

③ Sync

Saat online kembali, worker auto-flush ke API. Watermarking tetap dilakukan di server — klien tidak bisa memalsukan stamp.

PWA Installable di Android & iOS

  • Install banner otomatis di Android Chrome
  • Modal panduan "Add to Home Screen" untuk iOS
  • Service worker auto-update — bundle baru langsung dipakai
  • Web Push (VAPID) → notif review & fraud-alert real-time

Realtime & tenancy

Satu cabang, satu pandangan — tanpa polling

SSE Server-Sent Events

Invalidate cache di klien hanya saat ada perubahan nyata (kunjungan baru, review berubah, posisi GPS baru). Hemat baterai, hemat kuota, hemat DB.

RLT Row-Level Tenancy

Setiap query Prisma menyertakan branchId dari konteks user. SUPERVISOR cabang A tidak bisa melihat data cabang B — diuji di integration test.

HQ ADMIN HQ switch

ADMIN HQ dapat berpindah cabang via header x-branch-id. Setiap perpindahan ter-audit, termasuk siapa, kapan, dari mana.

CRT Concurrency-safe

BigInt-safe nominal rupiah, optimistic locking di reassign nasabah, transaksi atomik untuk auto-balance round-robin.

Dashboard supervisor — KPI & pergerakan angsuran live
